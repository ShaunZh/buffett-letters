# 公司研究指南

本文档定义了为 buffett-letters 项目生成公司投资案例研究文章的标准流程。阅读本文档后，应能够独立完成从选择公司、检索资料、撰写文章到输出文件的全过程。

---

## 一、任务目标

为 `src/content/companies/` 目录下的公司存根文件（只有 frontmatter + "待补充"）撰写完整的中文投资案例研究文章，格式参考 [Commonwealth Trust Co. 案例研究](../src/content/companies/commonwealth-trust-co-union-city-new-jersey.md)。

---

## 二、公司选择

### 2.1 识别待研究公司

1. 扫描 `src/content/companies/` 目录
2. 找出内容较少（通常只有 7 行左右，仅含 frontmatter + `（待补充）`）的存根文件
3. 优先选择在巴菲特致股东信中有明确提及、资料相对丰富的公司
4. 优先选择巴菲特早期投资案例（1950s-1970s），因为这些案例更具研究价值

### 2.2 需与用户确认

选定公司后，告知用户你的选择并说明理由。用户确认后再开始检索和撰写。

---

## 三、资料来源

### 3.1 首要来源：项目内巴菲特信件

项目包含丰富的巴菲特致股东信和合伙人信：

| 目录 | 路径 | 说明 |
|------|------|------|
| 伯克希尔致股东信 | `src/content/letters/` | 1971 年起，文件命名格式 `YYYY-letter.md` |
| 巴菲特合伙人信 | `src/content/partner-letters/` | 1957-1970 年，按日期命名 |
| 特殊信件 | `src/content/special-letters/` | 其他重要信件 |

**搜索技巧：**
- 使用 Grep 工具在 `src/content/letters/` 中搜索公司英文名或中文名
- 关键年份的信件要重点阅读（收购年份、后续回顾年份）
- 1985 年信有大量历史回顾，是重要来源
- 2004 年信有 NICO 承保纪律的完整表格
- 2021 年信有 55 年浮存金增长总结

### 3.2 次要来源：外部网络

优先查找一手资料：
- Berkshire Hathaway annual reports（berkshirehathaway.com）
- SEC filings
- Moody's Manuals / Poor's Manuals
- 报纸档案（New York Times 等）
- 州法院/公司注册/银行监管资料
- 公司年报或代理声明
- Wikipedia（作为起点，需交叉验证）

**注意：** 在当前环境下，外部网络可能被阻断（Wikipedia、berkshirehathaway.com 均可能出现连接超时）。此时应：
1. 坦白告知用户网络不通
2. 最大化利用项目内信件
3. 对于训练数据中的知识，明确标注"待复核"
4. 严格区分"已确认事实"和"训练知识"

### 3.3 来源可信度分级

| 级别 | 类型 | 处理方式 |
|------|------|---------|
| 一级 | 项目内信件直接引用 | 标注具体信件的 slug 引用 |
| 二级 | 训练数据中可追溯的事实 | 标注"待通过外部资料复核" |
| 三级 | 传记/二手资料 | 标注"来源为二手传记材料，读者应知悉" |
| 无法确认 | 记忆中不确定的数据 | 明确写"未能确认"或"资料不足" |

## 四、文章结构（必须遵循的模板）

按照以下 8 个章节组织文章：

```
# {公司名称}

## 1. 公司概况

## 2. 巴菲特收购/买入背景

## 3. 为什么买：投资逻辑

## 4. 交易详情与持仓规模

## 5. 收购后的业务发展与收益

## 6. 最终持有情况

## 7. 案例启示

## 8. 资料可信度与待复核点
```

### 4.1 各章节内容要求

**第一节：公司概况**
- 公司全称、所在地、行业、主营业务
- 成立时间、创始人（如有可靠来源）
- 被收购时的规模数据（保费、资产、收入等）
- 公司后来的命运（是否改名、合并、清算、仍存在）

**第二节：收购/买入背景**
- 巴菲特当时的处境和动机
- 收购的具体经过和关键故事
- 如涉及轶事（如 Ringwalt 的"15 分钟窗口"），须标注可信度

**第三节：投资逻辑**
- 属于哪类投资（低估值普通股、控股型、保险浮存金等）
- 安全边际在哪里
- 催化剂是什么
- 最好用表格总结投资逻辑维度

**第四节：交易详情**
- 必须用表格列出关键参数：收购时间、价格、规模、持股比例等
- 管理层的保留与传承情况
- 每个数字都要标注来源

**第五节：收益分析**
- 浮存金增长（如适用）
- 经营数据表格（如有）
- 收益率计算（如可计算）
- 定性分析（如无法定量）

**第六节：最终持有情况**
- 是否卖出、何时、卖给谁
- 卖出原因
- 或者为何一直持有

**第七节：案例启示**
- 体现巴菲特的什么投资特征
- 与 Graham 式投资、特殊情况投资等的关系
- 今天的投资者能学到什么
- 可以用表格总结

**第八节：资料可信度**
- 已确认事实列表
- 合理推测列表
- 仍需人工复核的信息列表
```

### 4.2 内部引用格式

**推荐使用"引用 + 引用链接"** 格式，以便正文引用可点击和易于区分。例如：

```markdown
巴菲特在 1985 年信中回顾道：

> "1967 年初，纺织业务产生的现金被用于通过收购国民赔偿公司进入保险业..."（[Berkshire Hathaway 1985 年致股东信][2]）
```

底部使用参考式链接，引用项目内部信件使用 Astro 路由:

```markdown
[1]: /letters/1977-letter/ "Berkshire Hathaway 1977 年致股东信"
[2]: /letters/1985-letter/ "Berkshire Hathaway 1985 年致股东信"
```

**重要**：引用链接应使用 Astro 路由而非文件路径：
- 正确：`/letters/1977-letter/`（对应 `src/content/letters/1977-letter.md`）
- 错误：`../letters/1977-letter.md`（这是文件系统路径，路由不正确）

**注意**：内容渲染容器为 `.company-detail__body` 和 `.rich-text`，链接样式已配置为 `color: var(--accent)` 并带有下划线。

## 五、写作规范

### 5.1 基本要求

1. **用中文输出**，结构清晰，使用二级标题和必要表格
2. **重点解释**"巴菲特为什么买、怎么赚钱、最后怎么处理"，不要只写公司简介
3. **每个关键数字、日期和结论都要给出来源链接或来源说明**
4. **区分事实、推测和基于资料的合理判断**
5. **如果资料不足，明确写"资料不足"或"未能确认"，不要编造**
6. **如果不同来源存在冲突，列出冲突并说明你认为哪一个更可靠**

### 5.2 引用规范

一级来源（项目内信件）引用格式：
```markdown
（[Berkshire Hathaway 1977 年致股东信][1]）
或短格式：
（[1977 年信][1]）
或列表中的格式：
- 1967 年初被收购（[1977 年信][1]、[2021 年信][4] 确认）
```

对于训练数据中的信息：
```markdown
（来源：公司创始年份来自巴菲特的公开表述和传记记载，**待通过 Nebraska 州公司注册记录复核**）
```

对于二手轶事：
```markdown
> **可信度备注**："15 分钟发怒窗口"的故事广泛流传于巴菲特传记和访谈中，但在巴菲特致股东信和伯克希尔年报的正文中未见直接记载。此处来源为二手传记材料，读者应知悉其轶事性质。
```

### 5.3 数字格式

- 金额：860 万美元 / $8.6 million
- 比率：约 57% / 7,700 倍
- 年份跨度：55 年间 / 1967—2004
- 范围：1,900—2,000 万美元

### 5.4 禁止事项

1. **禁止编造数据**：没有可靠来源的数字就用"未能确认"
2. **禁止把训练知识当作事实**：必须标注来源类型
3. **禁止混淆公司**：确认公司在不同时期的名称变化
4. **禁止忽略冲突来源**：如有矛盾，列出并分析
5. **禁止省略"待复核"标注**：所有非项目信件直接证实的信息都应标注
6. **禁止使用文件系统路径作为链接**：必须使用 Astro 路由格式

## 八、完整操作流程

1. **选公司**：扫描 `src/content/companies/`，找到存根文件
2. **读参考**：阅读 [Commonwealth Trust 案例](../src/content/companies/commonwealth-trust-co-union-city-new-jersey.md) 了解目标质量
3. **搜信件**：用 Grep 搜索项目内所有信件中该公司的提及
4. **读关键信**：重点阅读收购年份和回顾年份的信件
5. **尝试外部搜索**：如网络可用，搜索一手资料
6. **撰写文章**：按照第四节的结构模板编写，每个关键数字标注来源
7. **设链接**：底部使用 Astro 路由格式的参考链接
8. **标注可信度**：严格区分已确认事实、合理推测和待复核信息
9. **检查**：确认没有编造的数据，所有引用可追溯

## 九、经验教训（从本案例积累）

1. **外部网络不可靠**：Wikipedia、berkshirehathaway.com 都可能连接超时，必须有备用方案
2. **项目内信件是最可靠的来源**：1977 年信、1985 年信、2004 年信、2021 年信有大量可引用的历史回顾
3. **链接格式容易出错**：文件系统相对路径在 Astro 路由中不工作，必须用 `/letters/slug/` 格式
4. **参考式链接的可见性**：需要确保 CSS 为内容区链接设置了颜色和下划线，否则链接与普通文字无法区分
5. **轶事需标注**：著名的巴菲特故事（如"15 分钟窗口"）虽广为流传，但在致股东信中未见原文，应标注为二手来源
6. **数字要精确引用**：860 万收购价来自 1977 年信和 2021 年信双重确认，才有高可信度
